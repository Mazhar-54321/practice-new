# Takeaway — Next.js Day 001: SSR, SSG, CSR & Routing

- **Next.js solves both SSG and SSR — not just build time rendering**

- **Hydration = silent attachment of events to server rendered HTML in background**

- **SSR = rendered fresh on every request. SSG = rendered once at build time, served from CDN forever**

- **App Router = `app` folder + Server Components by default. Pages Router = `pages` folder + Client Components by default**

- **Using `"use client"` everywhere in Next.js defeats its purpose — use it only when interactivity is needed**

- **SSG = cook once serve forever. SSR = cook fresh every order**
